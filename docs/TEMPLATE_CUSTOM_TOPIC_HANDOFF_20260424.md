## Feature Changes

1. Business topic routing and labels:
   - `建联触达` is now `初次建联`.
   - Added `二次触达agency绑定`.
   - Added `待召回`.
   - Topic picker exposes topic groups and second-level intent labels.

2. Custom topic templates:
   - Added `custom_topic_templates` schema with `media_items_json`.
   - Added `server/routes/customTopicTemplates.js`.
   - Mounted routes in `server/index.cjs`.
   - APIs:
     - `GET /api/custom-topic-templates`
     - `POST /api/custom-topic-templates`
     - `PUT /api/custom-topic-templates/:id`
   - Saving a template stores it under the current topic classification (`topic_group`, `intent_key`, `scene_key`).
   - Updating a saved template supports renaming and replacing template text.

3. Template card UI:
   - Removed the composer-level `编辑模板` toolbar panel.
   - Template operations now live inside each template card.
   - `编辑模板` loads the template text into the chat composer only.
   - `保存模板` opens a centered large editor for creating a custom template.
   - `更新模板` opens the same centered large editor for updating the saved template row in the database.
   - The editor has a right-side topic route panel with all first-level topics and second-level subtopics.
   - Saved templates appear in the topic dropdown under `已保存模板`.

4. Reply Deck sizing:
   - Desktop Reply Deck has a top drag handle.
   - Operators can drag the handle to resize the candidate panel height.

5. Template images:
   - Custom templates can store `media_items` / `media_items_json`.
   - The card editor accepts corresponding image URLs, one per line, including local public asset paths such as `/sop-assets/apr-2026/image1.png`.
   - Template cards render saved image previews even when the template text is empty.
   - SOP image buttons in the topic route panel create image-only templates; their previous summary text is no longer bound to the image.
   - Backend template retrieval no longer auto-infers SOP images for text templates. If an operator needs an image, save/select the corresponding image-only template.
   - Images are now independent sendable assets. Operators send template text with `直接发送` and send each image with the image tile's `单独发送图片` action.
   - This supports follow-up conversations where a template uses referenced document/SOP images as separate WhatsApp media messages.

## Known Limits

- Image support currently stores image URL metadata; it does not upload files from the card editor.
- Media-only templates are valid as long as they have a label and at least one image URL.
- `更新模板` requires a saved custom template id. Built-in SOP/RAG templates can be saved as custom templates first, then updated.
- Redis may be unavailable locally; cache errors are non-blocking.
- `public/assets` and Vite caches may be regenerated by the next build.
