## Feature Changes

1. Business topic routing and labels:
   - `建联触达` is now `初次建联`.
   - Added `二次触达agency绑定`.
   - Added `待召回`.
   - Topic picker exposes topic groups and second-level intent labels.

2. Custom topic templates:
   - Added `custom_topic_templates` schema with `media_items_json`.
   - Added `server/routes/customTopicTemplates.js`.
   - Mounted routes in `server/index.cjs`.
   - APIs:
     - `GET /api/custom-topic-templates`
     - `POST /api/custom-topic-templates`
     - `PUT /api/custom-topic-templates/:id`
   - Saving a template stores it under the current topic classification (`topic_group`, `intent_key`, `scene_key`).
   - Updating a saved template supports renaming and replacing template text.

3. Template card UI:
   - Removed the composer-level `编辑模板` toolbar panel.
   - Template operations now live inside each template card.
   - `编辑模板` loads the template text into the chat composer only.
   - `保存模板` opens the attached card form and creates a custom template in the current topic classification.
   - `更新模板` opens the attached card form and updates the saved template row in the database.
   - Saved templates appear in the topic dropdown under `已保存模板`.

4. Template images:
   - Custom templates can store `media_items` / `media_items_json`.
   - The card editor accepts corresponding image URLs, one per line.
   - Template cards render saved image previews under the matching template text.
   - This supports follow-up conversations where a template must include images from the referenced document/SOP.

## Known Limits

- Image support currently stores image URL metadata; it does not upload files from the card editor.
- `更新模板` requires a saved custom template id. Built-in SOP/RAG templates can be saved as custom templates first, then updated.
- Redis may be unavailable locally; cache errors are non-blocking.
- `public/assets` and Vite caches may be regenerated by the next build.
